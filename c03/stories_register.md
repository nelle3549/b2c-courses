## Build notes

- 13-step flow identical to c02; artifact = **the full Value Proposition Canvas** (Nelle's revision, 2026-07-09): 9 fields in flow order — f1 customer segment · f2 customer jobs ×3 · f3 top-3 pains ×3 · f4 top-3 gains ×3 · f5 pain relievers ×3 · f6 gain creators ×3 · f7 products &amp; services ×3 · f8 value proposition · f9 commitment. The old 7-field Mini-VPC set (with a stored customer-phrases field) is superseded; the phrases proof now lives in the f8 trigger guidance.
- Field 7 (products &amp; services) is a **single statement field** (Nelle's call, 2026-07-09): the box names what you ship, it is not another ranked list. Fields with 1:1 mapping logic (jobs, pains, gains, relievers, creators) stay as per-item inputs.
- **Echo panels** (`data-af-echo` in interact.js): the reliever inputs display the learner's saved pains, creators display gains, and products &amp; services displays both relievers and creators, live-updating, so 1:1 mapping happens against their own words.
- **Compiled output uses the poster arrangement** (`.vpc-*` in v3_addons.css): Value Proposition + Customer Segment strips on top, value map square (gain creators / products &amp; services / pain relievers, 6px corners) on the LEFT, customer profile circle (gains / customer jobs / pains, 22px corners) on the RIGHT; commitment as footer row; single column on phones; same layout on the s12 result screen.
- Bank: 15 items per the v3.1 plan (b13 options reordered so keys spread 3-4-4-4), per-option explanations, no em-dashes; `c03_bank.json` generated from `c03_bank.js`.
- Diagrams: ranking matrix (s03) and 1:1 map (s05) as theme-v3 SVGs; the canvas itself stays the official bitmap.
